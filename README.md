# miniguia-estudos-notebooklm
Projeto desenvolvido durante o desafio da DIO utilizando NotebookLM para organização de estudos sobre Cybersecurity.
📚 Caderno Temático - Cybersecurity com NotebookLM
Objetivo

Estudar os fundamentos da Cybersecurity utilizando o NotebookLM para organizar informações provenientes de livros, artigos e vídeos, criando um material de revisão reutilizável.

1. Contexto

Explicar por que escolheu Cybersecurity.

Exemplo:

A área de Cybersecurity está entre as profissões mais promissoras da tecnologia. O objetivo deste estudo foi compreender os principais conceitos da segurança da informação, pentest, redes de computadores e boas práticas de defesa utilizando o NotebookLM como ferramenta de apoio aos estudos.

4. Engenharia de Prompts

Aqui é onde a maioria das pessoas perde pontos.

Não basta colocar perguntas.

Você deve mostrar seu raciocínio.

Exemplo:

Prompt 1

Explique Cybersecurity como se eu fosse um iniciante.

Resultado

A IA apresentou uma visão geral sobre segurança digital, explicando confidencialidade, integridade e disponibilidade.

Problema encontrado

A resposta ficou superficial.

Novo Prompt

Explique Cybersecurity para alguém que deseja trabalhar como Analista de Segurança e inclua exemplos práticos.

Resultado

Resposta muito mais completa.

Prompt 2

Explique todas as fases de um Pentest.

Resultado

A IA apresentou:

Reconhecimento
Enumeração
Exploração
Pós-exploração
Relatório

Problema

Poucos exemplos.

Novo Prompt

Explique cada fase utilizando ferramentas do Kali Linux.

Resultado

Foram incluídas ferramentas como:

Gobuster

Burp Suite
Prompt 3

Explique a diferença entre Vulnerabilidade, Exploit e Payload.

Resultado

A IA explicou corretamente.

Problema

A resposta não possuía exemplos.

Novo Prompt

Utilize uma analogia do mundo real.

Resultado

A compreensão ficou muito melhor.

Foi gerado um roadmap organizado.

7. Glossário
Termo	Definição
Pentest	Teste autorizado para identificar vulnerabilidades em sistemas.
Vulnerabilidade	Falha que pode ser explorada por um atacante.
Exploit	Código ou técnica utilizada para explorar uma vulnerabilidade.
Payload	Parte do ataque responsável por executar a ação maliciosa após a exploração.
Malware	Software desenvolvido para causar danos ou comprometer sistemas.
Firewall	Sistema que controla e filtra o tráfego de rede com base em regras de segurança.
IDS	Sistema de Detecção de Intrusões que monitora atividades suspeitas.
IPS	Sistema de Prevenção de Intrusões que bloqueia ataques automaticamente.
Criptografia	Processo de proteger informações por meio de algoritmos matemáticos.
Hash	Função que gera uma representação única de dados para verificar sua integridade.
